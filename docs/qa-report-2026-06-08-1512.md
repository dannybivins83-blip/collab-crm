## 4. Lead-intake automation + phone integration (Part C)

### Built + verified this run (safe prototype)
**`modules/lead_intake.py`** — pure, unit-tested parser/normalizer:
- `parse_email(sender, subject, body)` → `{name, phone, email, address, work_type, source}` with source auto-detect (AccuLynx / Angi / Thumbtack / Craigslist / Facebook / Email). Handles `Label: value` emails (AccuLynx, web-to-lead) first, then free-text regex fallback for phone/email/address, then a subject-line name fallback. Address regex is line-bounded so it can't bleed into the next field.
- `normalize(payload)` for structured web-form / webhook JSON; `clean_phone()` → `(xxx) xxx-xxxx`.

**Two token-guarded public webhooks in `modules/leads.py`** (registered in the auth PUBLIC allowlist):
- `POST /leads/intake` — one normalized JSON lead.
- `POST /leads/intake/email` — raw `{from, subject, body}`, parsed server-side.
- Both run the **same enrichment as a hand-keyed lead**: dedupe (by email or name+phone) → create contact + lead → resolve AHJ + roof system → **auto-draft starter estimate**. **Draft-only — never emails anyone.**
- **Auth:** gated by env `CRM_INTAKE_TOKEN` (header `X-Intake-Token` or `?token=`). **Fail-closed:** unset → `503 intake disabled`; wrong token → `403`.

**Live test results:**
- No token → `403 bad token` ✅
- AccuLynx email → parsed (name/phone/address/work type), created **lead 63** with AHJ, `system: metal`, auto-estimate **EST-0013** ✅
- Repeat POST → `created:false`, same lead 63 (dedupe) ✅

**Click-to-call (`tel:`)** added everywhere a phone shows: leads list, jobs list (as `.telcall` spans that don't trigger the row link), and the lead/job/contact detail cards (with the phone icon). Dashboard follow-up "Call" button already used `tel:`.
