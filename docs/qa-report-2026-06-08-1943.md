## Build-list items completed this run

### §4 "Ping Central" → **RingCentral inbound webhook** (call linking + call-channel intake)
- New `POST /leads/intake/ringcentral` (token-guarded, in the PUBLIC allowlist).
  - **Subscription handshake:** echoes RingCentral's `Validation-Token` header (200) so the webhook can be registered.
  - **Inbound call / voicemail** → parsed by `lead_intake.parse_ringcentral()` (tolerant of RingCentral's telephony-session and voicemail/message shapes; pulls caller ID + name).
  - **Call linking:** matches the caller by last-10-digits of phone → logs a **call activity** on the existing lead (or contact) timeline. Unknown caller → creates a new lead (`source: RingCentral Call/VM`) and logs the call. Outbound + non-call events are acknowledged and ignored (no dupes, no RC retries).
- **Verified live (local):** validation handshake echoes token ✅; new caller → lead 64 created ✅; call from lead 63's number → linked to lead 63 + "Inbound Call from (239) 555-3344" on its timeline (no duplicate) ✅; no token → 403 ✅.
- *My read on the open question:* **"Ping central" = RingCentral.** Flagging for confirmation — if it's a different platform, the same webhook shape adapts easily.
