# CRM Security & QA Audit — 2026-06-10

Multi-agent read-only audit of the white-label CRM (Flask, ~40 blueprints). Four reviewers
covered, in parallel: (A) auth/HMAC/portal security, (B) data layer + money math,
(C) integrations + sync bridges, (D) whole-app robustness + lifecycle process. Several
findings were **confirmed live** against throwaway local SQLite instances, not just reasoned
from source. Each finding cites `file:line`. `⊕` = independently found by more than one
reviewer (high confidence).

**Overall health:** core logic is solid — lifecycle, portal data model, e-sign safety, and
billing math were all checked and verified correct. But there is a cluster of **critical
access-control and money-parsing bugs** that must be fixed before this fully replaces AccuLynx.

## 🔴 CRITICAL (fix before any production cutover)

### 1. ⊕ Unauthenticated write endpoints — anyone on the internet can create/overwrite/wipe data
`/sync/browser-import`, `/leads/import`, and the whole `/sync/*` family (docs, photos, billing,
estimates, comms, **catalog-wipe**, `debug-probe`) are on the no-login `PUBLIC` allowlist, CORS
`*`, with **zero auth/signature**. Confirmed live: an anonymous client *created* a lead
(`added:1`) then *mutated* it (`updated:1`). `/sync/catalog-import` with `[]` **empties the
material catalog**.
- `modules/auth.py:24-30` (PUBLIC allowlist), `modules/acculynx_sync.py:856` (`browser_import`),
  `modules/leads.py:588` (`import_leads`), catalog `DELETE` at `acculynx_sync.py:1138`.
- **Fix:** ADD auth, don't just delist — these back the bookmarklet sync suite. Gate them with a
  shared-secret / `X-Signature` check mirroring how `measurements.ingest` / `takeoff.create`
  already verify. Remove `debug_probe` (its own docstring says "remove after use").

### 2. ⊕ Forgeable HMAC via public fallback secret — every signed seam
With `MEASURE_CRM_WEBHOOK_SECRET` unset (current live state on both hosts), the secret derives to
the publicly-documented `seabreeze-webhook-secret`. Confirmed live: a forged signature created a
job + estimate via `/api/takeoff`. Same secret mints SSO assertions + SiteCam links. The HMAC
math itself is correct (`hmac.compare_digest`, signs exact raw body) — the *fallback* negates it.
- `modules/measurements.py:228-237` (`_ingest_secret`), `modules/sso.py:116-119` (`_secret`).
- **Fix (two halves):** (config) set a strong `MEASURE_CRM_WEBHOOK_SECRET` on both hosts + the
  engine VM, byte-identical; (code) make verification **refuse** when the secret is unset in a
  non-dev environment instead of deriving a constant from the tenant slug. Do both in one pass so
  it can't silently regress.

### 3. `theme.est_num()` silently corrupts currency
`re.sub(r"[^0-9.]", "", ...)` strips the sign and leaves multiple dots: `-$500` → `500` (sign
lost), `($500)` → `+500` (credit flips positive), `$1.234.50` → `0.0` (float() fails → silent 0).
Every balance / collected / contract-value calc funnels through it.
- `theme.py:50-58`.
- **Fix:** treat leading `-` / surrounding `()` as negative; collapse multi-dot to the first
  decimal instead of returning 0.

## 🟠 HIGH

### 4. ⊕ Invoice & estimate numbers collide after a delete
`_next_number()` = `"INV-%04d" % (max_id + 1)`. Delete the latest invoice and the next insert
**reuses** the number (reproduced: two different invoices both `INV-0003`). Also racy under
multi-worker. Same defect in estimates.
- `modules/invoices.py:24-26`, `modules/estimates.py:64-66`.
- **Fix:** dedicated counter/sequence or `MAX(number)`-based with a UNIQUE constraint + retry;
  don't derive a human-facing id from a row id.

### 5. Hardcoded default `SECRET_KEY`
`SECRET_KEY = os.environ.get("CRM_SECRET", "white-label-crm-dev-secret")` → forgeable admin
session cookies if `CRM_SECRET` is unset in production.
- `config.py:41`.
- **Fix:** fail to boot (or loud-warn) when `CRM_SECRET` is unset in non-dev. Verify it's set on Render.

### 6. SSRF via `report_url` in measurement ingest
After signature passes, `urllib.request.urlopen(body["report_url"], timeout=30)` fetches an
arbitrary attacker URL server-side (e.g. cloud metadata `169.254.169.254`). Compounds with #2.
- `modules/measurements.py:320-326`.
- **Fix:** allow-list scheme (`https`) + host (the measurement app origin); block
  loopback/link-local/RFC1918; disable redirects. Or require `pdf_base64`/multipart only.

### 7. No CSRF protection app-wide + no SameSite/Secure cookie flags
Authenticated admin POSTs (user/role/password at `settings.py`, estimate sign, deletes) are
triggerable cross-site. No Flask-WTF, no per-form token, no `SESSION_COOKIE_*` config.
- app-wide; e.g. `settings.py:54`, `estimates.py:336`.
- **Fix:** set `SESSION_COOKIE_SAMESITE="Lax"`, `SECURE=True`, `HTTPONLY=True`; add CSRF tokens on
  authenticated state-changing routes.

### 8. Checklist gating does not gate
`set_stage` / `advance` / `move` validate only that the target stage key exists — none read the
stage checklist. A job can jump `approved → closed`, skipping permits/inspections. Checklists are
display-only.
- `modules/jobs.py:235-273`, `modules/leads.py:321-341`.
- **Fix:** refuse/warn-with-override when the current stage's checks are incomplete, or document
  that checklists are advisory.

### 9. Takeoff ingest: three related data-integrity bugs
- **Multiple measurements overwrite each other** — upsert-by-job inside a loop keeps only the
  last; `measurement_ids` returns the same id repeated. `modules/takeoff.py:126-140`.
- **Idempotency is O(n) scan + TOCTOU race** — loads all `takeoffs` rows, no UNIQUE index; two
  concurrent identical POSTs both create. `modules/takeoff.py:85-89`.
- **Substring address match attaches to the WRONG job** — `"1 Main St"` matches `"11 Main St"`,
  and "newest job wins" (scans `id DESC`). `modules/measurements.py:270-278`.
- **Fix:** insert each measurement (don't upsert-in-loop); UNIQUE index on `idempotency_key` +
  insert-first-catch-violation; exact-address match first, refuse on ambiguous multi-hit.

### 10. Sync cursor can skip or double-import rows
Resumable cursor stores an absolute offset into an AccuLynx list sorted `MilestoneDate Desc`; any
milestone edit between clicks shifts rows under the cursor → re-import or skip. Billing writes on
every hit, so doubles = double payments. Off-by-one at the capped-group boundary too.
- `modules/acculynx_sync.py:2188-2233`.
- **Fix:** page by a stable key (job GUID / createdOn ascending), or de-dup by GUID across the
  whole walk (billing isn't currently de-duped).

### 11. Automation/notification engine fails silently & can double-fire
Recovers the stage by regex-parsing the English string `"Moved to X"`, wrapped in
`except: pass`; `db.add_activity` is monkey-patched globally. Reword a label → notifications
vanish with no trace; two stage activities for one transition → fires twice.
- `modules/automations.py:30-35, 74-84, 146-152`.
- **Fix:** pass the stage key explicitly instead of parsing prose; log on exception; dedupe per
  (entity, stage, day).

### 12. File uploads: no type/MIME/size guard, then served back
`upload_photo`/`upload_doc` accept any file and `f.save()` it; later served from `/uploads/<path>`.
An uploaded `.html`/`.svg` is stored and served (stored-XSS surface). Filename is sanitized
(no traversal), but content type is unchecked.
- `modules/files.py:30-60` + `app.py:95`.
- **Fix:** allow-list extensions/MIME per kind; serve user uploads with
  `Content-Disposition: attachment` or a fixed safe content-type.
