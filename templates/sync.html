🧭 Records & Milestones → CRM — no API key needed

Pulls leads/jobs straight from the AccuLynx page you're logged into (uses your browser session, not the API). Now captures the EXACT milestone of whatever list you open — so the CRM stage matches AccuLynx 1:1 (Documentation Needed, Permit Applied For, Tear Off Started, etc.), not just the top-level bucket.

  1. One-time setup: drag this button to your bookmarks bar → 📥 AccuLynx Records → CRM
  2. In AccuLynx, click a Milestone Pipeline filter on the left (e.g. Approved, Permit Applied For, Invoiced) so the URL shows that milestone.
  3. Click the bookmark once. It auto-scrolls the whole list, shows a live "Collecting…" counter, and imports everyone at that exact milestone. Run it once per milestone list to get every stage right — duplicates are skipped/updated.
Tip: to fix mismatched stages, walk each granular milestone list once. If your browser blocks the cross-site request the first time, allow it — it's just posting to your own CRM at {{ request.host_url }}.

⭐ Sync Everything → CRM (one click)

The closest thing to a "do it all" button. In one click on a Jobs list, it pulls each job's billing & payments, estimates, and communications from your logged-in AccuLynx session into the matching CRM job. (Permit documents stay on their own button below — they're large file transfers, so they run best separately.)

Why this is a bookmark, not the 🔄 Sync button: the Sync button runs on the server via AccuLynx's public API, which only returns leads/jobs/stages (it 404s on money, estimates & messages). That data lives behind your browser login, so it can only be pulled by a bookmark running inside an AccuLynx tab.

  1. One-time setup: drag to your bookmarks bar → ⭐ AccuLynx Sync All → CRM
  2. Run a regular 🔄 Sync now first so the jobs exist, then open a Jobs milestone list (Approved / Completed / Invoiced hold the money).
  3. Click the bookmark once. Banner shows "All X/Y · N valued · E est · M msgs". Safe to re-run — everything dedups.

📎 Pull permit documents into jobs

Attaches each job's permit-related files (Permit, NOC, HOA Approval, Certificate of Completion, inspection) from AccuLynx to the matching CRM job. Files come from your logged-in AccuLynx session (the API can't serve documents). Already-attached files are skipped; large scans upload in chunks.

  1. One-time setup: drag to your bookmarks bar → 📎 AccuLynx Permits → CRM
  2. In AccuLynx, open a Jobs milestone list — Completed first, then Invoiced (sort by date / scroll to the range you want).
  3. Click the bookmark once. It scrolls to load the list, then pulls every job's permit docs with a live progress banner. Safe to re-run — it only adds what's missing.
Tip: run it per milestone list. The job must already be synced into the CRM (run a sync first) or it'll be counted as "not in CRM".

📐 Pull roof reports into measurements — 20 at a time

Pulls each job's RoofGraf / roof-report PDF from AccuLynx job documents into the matching CRM job or lead, attaches it under Measurements, and auto-parses it — squares, pitch, ridge/hip/valley/eave/rake — so it feeds the estimate. The server walks your active pipeline (Assigned → Prospect → Approved → Completed → Invoiced → last 500 Closed) with a resumable cursor, handing the collector 20 jobs per click; click again to continue where it left off. Jobs that already have a roof report are skipped (idempotent).

  1. One-time setup: drag to your bookmarks bar → 📐 AccuLynx Roof Reports → CRM
  2. Make sure an API key is saved above (the server uses it to enumerate the next 20 jobs) and you've run a 🔄 Sync so the jobs/leads exist in the CRM.
  3. In any logged-in my.acculynx.com tab, click the bookmark. It asks the CRM for the next 20 jobs, pulls each one's roof report, parses it, and shows a live banner. Click it again to walk the next 20 — the banner says when a full pass is complete.

💵 Pull billing & payments into jobs

Pulls each job's job/contract value, invoices, payments & balance from your logged-in AccuLynx session into the matching CRM job. This is what fixes the dashboard L/P/A/C/I dollar totals — the public jobs API has no money field, so values only arrive via this bridge. Re-running updates the same records (deduped by AccuLynx id); never duplicates.

  1. One-time setup: drag to your bookmarks bar → 💵 AccuLynx Billing → CRM
  2. In AccuLynx, open a Jobs milestone list (Approved / Completed / Invoiced hold the money).
  3. Click the bookmark once. It scrolls the list, fetches each job's financials, and posts them with a live banner. Safe to re-run.
Job must already be synced (run a Sync first). Confirm field mapping any time with 🔬 Internal API test.

🧾 Pull estimate LINE ITEMS (exact AccuLynx copy)

Goes beyond the estimate header — pulls every line item of each AccuLynx estimate (description, qty, unit, cost & price, grouped by section) and stores them as the CRM estimate's lines. This is the 100% exact mirror of an existing AccuLynx estimate (vs. the calibrated template used for brand-new estimates). Re-running replaces the lines in place (idempotent).

  1. One-time setup: drag to your bookmarks bar → 🧾 AccuLynx Estimate Lines → CRM
  2. In AccuLynx, open a Jobs milestone list (the jobs whose estimates you want copied).
  3. Click the bookmark once. It scrolls the list, pulls each job's estimate line items, and posts them with a live banner. Safe to re-run — lines are replaced, not duplicated.
Job must already be synced (run a Sync first). Pairs with the calibrated templates: this mirrors existing estimates exactly; the templates build new ones.

💬 Pull communications (notes & messages) into jobs

Pulls each job's messages, notes & comments (body, author, date, type) into the CRM as job activities + the record narrative. High volume — capped at 200 newest per job per run; capped counts are reported. Deduped so re-runs don't pile up.

  1. One-time setup: drag to your bookmarks bar → 💬 AccuLynx Comms → CRM
  2. In AccuLynx, open a Jobs milestone list (start with a small one — this is the heaviest pull).
  3. Click the bookmark once. It scrolls, fetches each job's thread, and posts it with a live banner. One slow job won't stall the rest.
